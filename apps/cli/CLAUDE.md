# apps/cli — the `brain` CLI (`@atlas/cli`)

The single Atlas application, and in v2 the **only** process. `package.json` maps `bin.brain → dist/bin.js`; ESM (`type: module`), build `tsc -p tsconfig.json`, test `vitest run --passWithNoTests`. Depends (runtime) on the survivor workspace packages — `@atlas/contracts|git|jobs|lancedb-index|models|sources|sqlite-store` + `yaml`/`zod`; `@atlas/testing` is a devDependency. (`@atlas/broker` + `@atlas/scan` are DELETED — ADR-0003.) Two things live here: the **command surface** (thin plumbing under `src/{bin,main,router,handlers,index}.ts` + `src/{commands,config,render,errors,diag}/`) and the **engine** (`src/{workflows,policies,trust,synthesis,markdown,vault,validation,retrieval,ingest,sync,locks}/`) every mutating command drives through.

**v2 posture (ADR-0003).** `brain` opens the vault working tree + SQLite projection + LanceDB index, mutates notes, commits to git, exits. **Git is the only safety mechanism** — one commit per applied ChangePlan on `refs/heads/main` is the audit trail AND the undo (`git revert <sha>` + `brain sync`). No brokers, no daemons, no OS identities, no scan, no quarantine, no trust tiers, no audit ledger, no authorization challenges — retired in place, revivable only from the `v1-fortress` tag.

## Command surface (plumbing)

- **`bin.ts`** — `#!/usr/bin/env node`, `void main()`. No logic. **`main.ts`** — `runCli(argv, env, options)` (the DI'able entry that **never throws**) + `main()` (process launcher, last-resort `try/catch` → `internal` envelope exit 4, kept separate so it survives a symlinked bin).
- **`router.ts`** — `loadRegistry` (reads [`commands.json`](../../docs/specs/cli-contract/commands.json)), `parseArgv`, `sniffOutputFlags`, `matchCommand`. Command name = the **longest** registry entry prefixing the leading non-flag run (`evidence resolve` matches as two tokens, not `evidence`; `note related` beats a shorter `note` prefix); everything else forwarded in **original order**. `--` = verbatim passthrough boundary. Globals stripped anywhere: `--json --plain --no-color --quiet/-q --verbose/-v --help/-h --config <path>`.
- **`handlers.ts`** — `RunContext`, `HANDLERS` map, `registerCommand`. **Deliberately a leaf module, not in `main.ts`**: command modules `registerCommand(...)` at import time and `main.ts` side-effect-imports `commands/index.js`; if `HANDLERS` lived in `main.ts` that import would hit a temporal-dead-zone crash.
- **`commands/index.ts`** — the registration barrel: **20** side-effect imports registering the **24** v2 survivor commands (#333; jobs/source/note each register several; `link` is the one NEW v2 command, #331). `commands/pagination.ts` = the shared pagination contract (`DEFAULT_LIMIT=50`, `MAX_LIMIT=500`, strict `INT_LEXICAL_RE`, `assertOffsetInRange`). `commands/store-open.ts` = `openMigratedStore` (read paths, never migrates) + `registerFeatureMigrations`.
- **`commands/job-handlers.ts`** — the production workflow→executor registry. **EMPTY in v2 (task 4-4):** `reverify`, the retention sweeps, trust remediation, and the absorb-cycle `index:reconcile` all died with their enqueuers (ADR-0003 / #334); `evidence retry` is now a synchronous frontmatter mutation, not a job. `PRODUCTION_WORKFLOWS = []`, `buildJobHandlers` returns `{}`, and the completeness/laziness gates (`test/jobs.registry-completeness.test.ts`) hold vacuously. The durable queue is retained for `jobs run|list` + the env-gated test handler (`jobs-test-handler.ts`) + any future workflow.
- **`config/{schema,load}.ts`** — Zod **strict** schema for `brain.config.yaml`. Owns every plan threshold; modules read `AtlasConfig`, never literals. `ConfigError` carries `exitCode=2` + `{file, key?}`. `retrieval.rrf.weights.vector` is `.gt(0)` (a zero vector weight annihilates the §6 FTS-fallback fusion); `fts` may be 0. `load.ts` applies `ATLAS_*` env overrides then strict-parses. The schema still declares **12 sections** including the now-**inert** `broker`/`quarantine` blocks (D8 `audit_anchor_path`, D10 sockets) — v1 residue **nothing in v2 consumes** (grep confirms they are referenced only in `schema.ts` itself); the config-schema demolition that drops them is pending. There is no `canonical-ref.ts` and no `git.canonical_ref` — canonical IS `refs/heads/main`, hard-coded (`workflows/mutation-order.ts` `CANONICAL_BRANCH`).
- **`render/safe.ts`** — `render()` is the single human-output path; `sanitize()` strips CSI/OSC (incl. OSC-8/OSC-52), lone `ESC`, C1, C0+DEL **except `\t`/`\n`** (CR IS stripped — kills CR-overwrite), and bidi controls. Colour-free by construction (`NO_COLOR`-clean). `render/plain.ts` `resolveOutputMode`; `render/progress.ts` append-only, **stderr** by default.
- **`errors/envelope.ts`** — `CliError` (single failure type) + `EXIT = {OK:0, VALIDATION:1, CONFIG:2, INTERNAL:4, USAGE:5}`. `emitJson` (NDJSON success), `emitError`. The retired security architecture's `SECRET_SCAN` (3) and `ACTION_REQUIRED` (6) codes were removed with the scan/trust demolition (ADR-0003).
- **`diag/logger.ts`** — JSONL logger, size-rotated on write (`logs.max_bytes`/`max_files`). Redaction boundary: `msg` restricted to a stable `EVENT_ID` token (else `[redacted-message]`); `redact()` scrubs `SENSITIVE_KEY`-matching context; reserved fields (`ts/level/runId/jobId/msg`) can't be spoofed.

### Exit codes (`0 ok · 1 validation · 2 config/vault/lock · 4 internal · 5 usage`)
The v2 set is `{0,1,2,4,5}`. **No single-error-envelope command emits process exit 7.** The contracts' nominal "provider-retryable (7)" rides as exit 4 **carrying `retryable:true` + optional `retryAfterMs` on the envelope** — retryability is a flag, not a code (`query.ts`, `index-ops.ts`). The **one** exception is the `jobs run` batch aggregate: `packages/jobs/src/runner.ts` `aggregateExit` maps a transient-but-exhausted item to 7 and `main()` `process.exit`s it unclamped.

## Commands (the 24-command v2 survivor set)

All `privilege: shared` — v2 has no privileged/broker-authorized command. `commands.json` version 2, every row `implemented:true`.

| Group | Commands |
|---|---|
| foundation | `db migrate` · `db rebuild` · `status` |
| jobs | `jobs list` · `jobs run` |
| ingest / sources | `ingest` · `source add` · `source list` · `source show` |
| notes | `note add` · `note show` · `note history` · `note related` |
| sync / query | `sync` · `query` · `validate` |
| synthesis / evidence | `enrich` · `maintain` · `link` · `evidence review` · `evidence retry` · `evidence resolve` |
| index | `index rebuild` · `index eval` |

- **`status`** (#332) is the ONE merged read surface — it absorbed the retired `doctor`, `db status`, `index status`, `sync status`. `--json` = `vault`/`db`/`index`/`sync` sub-objects + `checks` = exactly `vault-reachable`, `git-healthy`, `provider-key-present` (the non-throwing `hasGeminiApiKey` probe), `index-not-stale`, `migrations-current`. Exit **0 whenever a payload was produced** (a failed probe is data, `ok:false` still exits 0); **2 only when the vault/config is unresolvable**. Read-only: never creates the DB, never applies DDL.
- **`link`** (#331, NEW) — add/re-alias/remove an edge. `SetLink` (no `--predicate`) = a plain `[[wiki-link]]` in the source note BODY (`## Links` section); `CreateRelationship` (`--predicate`) = a typed edge in the source note's frontmatter `related:` list (markdown-derived). Every non-noop lands exactly one commit; `commit:null` IFF `noop`.
- **`sync`** = `reconcile` (see below). **`index rebuild`** absorbs the retired `index repair|status|verify`.

## Engine

The mutating path is uniform: **every** mutating command grounds a plan and applies it as **one direct commit onto `refs/heads/main`** through `workflows/mutation-order.ts` `runMutation`. There is no agent worktree, no broker CAS, no tier gate, no review-pending.

- **`workflows/`** —
  - **`mutation-order.ts` `runMutation`** is THE machinery every mutating command wraps (`note add`, `ingest --apply`, `link`, `evidence resolve/retry`, `source add`, and `enrich`/`maintain` synthesis). Binding order: `withVaultMutation` (vault lock) → **assert `HEAD == refs/heads/main`** (feature-branch/detached ⇒ exit 2, no mutation) → `validate` (structural ⇒ exit 1) → `ground(preApply)` (retrieval/normalize/plan) → **dirty-vault gate** (a dirty target/source note ⇒ exit 1) → capture touched-path **preimage** → `apply` (write working tree) → **`commitPaths(refs/heads/main)`** → refresh LanceDB index → refresh SQLite projection (advances `notes.content_hash` — the sync cursor). A throw in apply/commit **restores the preimage**; a failed index write aborts before `content_hash` advances, so the next `brain sync` heals it. **Dirty-vault doctrine:** reads + `sync` treat dirt as normal; a mutation tolerates unrelated working-tree dirt but fails grounding on a dirty note it edits/names (on-disk hash ≠ projection `content_hash`, OR an uncommitted git diff vs `HEAD`).
  - **`reconciler.ts`** — `reconcileRunsOnStartup`, the startup run-sweep. v2 (#338) rewrote it to a pure `agent_runs` sweep — the §2.8 audit-ledger drain + the un-anchored-integration barrier are retired; recovery re-derives `integrated` from authoritative git containment (git is the sole source of truth). No audit event on `failed@`; no backup gate on `finalized`.
  - **`direct-integrator.ts` `makeCanonicalIntegrator`** — the FF-CAS canonical-install seam (advance `refs/heads/main` under CAS; `@atlas/git`'s FF-only carve-out). No socket, no attestation key, no `refs/audit/runs`, no audit event.
  - **`ops/index.ts`** — `executeOp`/`isExecutableOp`; `EXECUTABLE_OPS` is currently **EMPTY** (`CreateRelationship`'s typed-`relationships:` executor is deferred; the v1 claims/evidence executors are retired, #337).
  - **Retained infrastructure, no v2 production caller:** the persisted run state-machine (`engine.ts` `startRun`/`RunHandle`, `checkpoints.ts`), `makeCanonicalIntegrator`, `model-plan-generator.ts`, and the reconciler's producer hooks are exercised by tests and kept for the future/legacy path — nothing in the v2 command surface enqueues a multi-checkpoint run (grep: no production `startRun(`/`reconcileRunsOnStartup`/`.integrate(` caller). Plus `synthesis.ts` (the plan pipeline), `rebuild-from-git.ts` (`db rebuild --from-git` DR), `idempotency.ts`, `run-report.ts`, `maintain.ts`.
- **`policies/`** — **`operation-gate.ts` is the ONE surviving gate** after the trust/scan demolition. `assertOperationAllowed`: reserved task ops (`CreateTask`/`UpdateTaskState`) ⇒ `reserved-operation` (exit 1); an op outside the contract set ⇒ `operation-forbidden` fail-closed (exit 1); every recognized synthesis op is permitted — **no tier gate**. (Risk/sensitivity/mutation-policy are gone.)
- **`trust/`** — DEAD residual read-only code (#334). The trust lifecycle (promote/revoke, taint, remediation) is retired; `readTrustRecord`/`trustStateFor` still read the `0010_trust_state` rows but have **no live caller** — `source list|show` no longer report trust (grep: `source.ts`/`source-add.ts` never call them). The `0010_trust_state` table + this directory are un-dropped residue; a follow-up cleanup removes them (not folded into any shipped migration).
- **`synthesis/`** — `model-output.ts`, the model-output orchestration boundary; the operation gate runs fail-closed before any sink. The synthesis apply path (`workflows/synthesis.ts`) is `retrieve → pack → generate ChangePlan → validate → patch → runMutation` — retrieval-first (an empty retrieval aborts before any plan), side-effect-free preview.
- **`retrieval/`** — `layers.ts` (identity short-circuit id→slug→alias, else fenced hybrid + RRF), `rrf.ts` (`Σ weight/(k+rank)`, `k=60`, UTF-8-byte tie-break), `pack.ts` (token-bounded), `wiring.ts`. Contract: [`retrieval-index-contract.md`](../../docs/specs/retrieval-index-contract.md).
- **`ingest/`** — `ingest.ts` (#340): takes a **source-registry id** (`source add`, #339), reads bytes at its `locator`, normalizes them through the surviving `@atlas/sources` `normalize()` **in-process** (no sandbox jail, **no secret scan** — ADR-0003 accepted residual risk), grounds a deterministic note (id from `(source.id, canonicalMediaType)` — idempotent upsert), commits it directly through `runMutation`, and stamps `source.lastIngestedAt` LAST. The normalized source text is emitted inside a fenced block so raw `[[wiki-links]]`/frontmatter are never parsed as vault structure.
- **`sync/`** — `diff.ts` (the pure `reconcile()` classification routine), `resolve-at-ref.ts` (`resolveAtRef(repo, CANONICAL_BRANCH, note_globs)` → `(noteId) → ParsedNote | null`, a leaf reader used by `link`/`enrich`/`maintain`/`evidence`). No absorb cycle, no cursor table, no canonical-ref indirection.
- Other engine dirs: `markdown/` (patch engine — `apply`/`patch`/`sections`/`parse`/`fence`/`diff-summary`; no whole-file rewrites), `vault/` (reader errors-as-values, temp-fsync-rename writer, `frontmatter`, `identity`, `note-matcher` = the glob filter for `note_globs`), `validation/` (`identity`/`provenance`/`accessibility`/`store-vault`/`vault-validate`), `locks/` (`manager.ts` = totally-ordered scopes ⇒ deadlock structurally impossible; `mutation-guard.ts` = `withVaultMutation`).

### brain sync (v2 reconcile — #329)

`sync` reconciles the vault **working tree** against the SQLite projection by **per-note content hash** — there is NO HEAD cursor; the projection's `content_hash` IS the cursor. Under the exclusive `vault-maintenance` lock it scans + classifies via the shared `readReconcile` routine into **changed/new/dropped/moved**, then:
- **stages** the reindex (embed + write + verify a new LanceDB generation) and **commits the projection fold + fenced activation atomically** (one SQLite txn) so the content-hash cursor advances IFF the verified generation goes live;
- **purges dropped invisibility-first** (SQLite row + identity keys + note_links + vault-derived evidence in one txn, then LanceDB vectors);
- **moves-in-place** pure moves (no re-embed) — a rename still folds so the path-derived slug identity key moves with it.

An unchanged tree is a structural **noop** with NO index write. `--dry-run` classifies but touches no sink. **Exit 0** (incl. noop), **2** on config/vault/lock (incl. an unreadable/structurally-broken vault — a duplicate stable id or unparseable in-glob note fails **closed**, nothing mutated), **4** internal, **5** usage. The same `readReconcile` routine feeds `status`'s four pending counts, so the two surfaces cannot disagree; `sync status` is retired (merged into `status`).

## Invariants & guardrails

- **Single human-output channel.** Only `render/safe.ts`, `render/progress.ts`, `errors/envelope.ts` may write stdout/stderr or `console.*` — enforced by `test/no-render-bypass.test.ts` (walks all of `src/`, allowlist pinned).
- **`runCli` never throws.** Every failure serialises (JSON) or renders + maps an exit code; `main()` is the last-resort guard (`test/main.test.ts`).
- **Terminal-injection safety is release-blocking** — `sanitize()` is the trust boundary before any byte hits a TTY.
- **`implemented:true` ⇒ handler registered** (`test/command-registration.test.ts`) — guards the #145 failure class (a typo'd/dropped registration merges green, dies live as `not-implemented`).
- **`db migrate` is the sole migration composition root.** It calls `store-open.ts` `registerFeatureMigrations(store)` *before* `store.migrate()`; every other command opens an already-migrated store via `openMigratedStore` and never applies DDL. Migrations run **through 0015**. v2 (#338): the §2.8 audit ledger + AEAD backup/watermark are retired — there is no `backup-unhealthy` write gate; git (one commit per ChangePlan on `refs/heads/main`) is the only safety mechanism, and `agent_runs`/`model_calls` are plain operational tables.
- **Canonical mutation order is fixed** (`runMutation`): a mutation can never precede validation + grounding, and `HEAD` must be `refs/heads/main` before any write. Preview commands (`ingest`/`enrich`/`maintain`) mutate only with `--apply`; `--dry-run` + `--apply` ⇒ usage exit 5.
- **The vault lock spans the whole mutation order.** Standalone mutating commands funnel through `locks/mutation-guard.ts` `withVaultMutation(ctx, vaultPath, fn)`, which holds `vault-maintenance` across grounding → apply → commit → refresh and threads a `preApply` callback that re-checks for an external git `index.lock` at the true post-grounding boundary (every synthesis retry). `sync`, `db migrate|rebuild`, `index rebuild`, `jobs run` take their scope directly via `ctx.withLock`; **`jobs run` takes `jobs-runner` OUTERMOST** (job handlers must NOT re-take `vault-maintenance`). **Read/preview commands never take a lock** — preview branches open the store through the non-migrating `openMigratedStore` (read-only, never creates the DB nor applies DDL). A writer that cannot take the lock exits 2 (`locked:<scope>`, no queueing); a pre-existing **external** git `index.lock` is a distinct preflight exit 2 (`git-index-locked`).
- **`ATLAS_ROOT`** overrides the auto-detected cli-contract root; precedence: `options.root` → `env.ATLAS_ROOT` → `findRoot` (walks up for `docs/specs/cli-contract/commands.json`).

## Gotchas & sharp edges

- **`sniffOutputFlags` mirrors `parseArgv`'s global-flag spelling** — a non-fallible second copy run *before* the fallible parse so a usage error still honours `--json`. A new global flag must land in **both** or the pre-parse-error path won't honour it.
- **`--idempotency-key` is parsed-but-unused by some commands** (`enrich`/`maintain` consume the flag+value but don't thread it); `jobs`/`source add`/`note add`/`ingest` DO honour it via the persisted caller-idempotency layer.
- **`coerceLike` passes bad env-override values through** to let the schema reject them with a clear message (`ATLAS_INDEXING_DIMENSIONS=foo` fails as a schema error, not silent coercion). Nested keys resolve longest-known-prefix-first; array overrides split on commas.
- **Installed-binary root resolution is layout-dependent** — a packaged install without the repo layout needs `ATLAS_ROOT` set or `loadRegistry` fails `internal`. Personal-project posture: run from the repo.
- **`indexNotes` consumes a `ParsedNote` PROVIDER, never fences.** `IndexDeps.notes` is `() => ParsedNote[]` — re-embedding needs the note BODY to chunk + embed, and `indexNotes` infers a REMOVAL from a requested id being **absent** from `deps.notes()`. `sync`'s dropped-note vector delete drives this with an EMPTY provider (`notes: () => []`) so the requested ids' chunks are dropped.
- **Engine `round-2`/`round-3 finding` comments are load-bearing** — CAS/ordering fixes closing real crash-safety holes from the #74 review; don't "simplify" `expectedFrom`/`assertAdvanced`/`expectChanges:1` away.
- **`jobs run` is the sole exception to the single error envelope** — it emits `{command, items[], aggregate}` even on partial failure, aggregate `exitCode` derived from the batch (the only path to exit 7). `jobs list` is read-only.
