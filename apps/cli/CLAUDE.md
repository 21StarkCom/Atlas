# apps/cli — the `brain` CLI (`@atlas/cli`)

The single Atlas application. `package.json` maps `bin.brain → dist/bin.js`; ESM (`type: module`), build `tsc -p tsconfig.json`, test `vitest run --passWithNoTests`. Depends (runtime) on **every** workspace package (`@atlas/broker|contracts|git|jobs|lancedb-index|models|scan|sources|sqlite-store`) + `yaml`/`zod`; `@atlas/testing` is a devDependency. Two things live here: the **command surface** (thin plumbing under `src/{bin,main,router,handlers,index}.ts` + `src/{commands,config,render,errors,diag}/`) and the **engine** (`src/{workflows,policies,trust,graduation,synthesis,markdown,vault,validation,retrieval,ingest,quarantine,purge,locks,retention,audit,broker}/`) every mutating command drives through.

Repo constitution: [`../../CLAUDE.md`](../../CLAUDE.md). Design SSOT: [`../../docs/specs/2026-07-11-atlas-v1-design.md`](../../docs/specs/2026-07-11-atlas-v1-design.md).

## Command surface (plumbing)

- **`bin.ts`** — `#!/usr/bin/env node`, `void main()`. No logic. **`main.ts`** — `runCli(argv, env, options)` (the DI'able entry that **never throws**) + `main()` (process launcher, last-resort `try/catch` → `internal` envelope exit 4, kept separate so it survives a symlinked bin).
- **`router.ts`** — `loadRegistry` (reads [`commands.json`](../../docs/specs/cli-contract/commands.json)), `parseArgv`, `sniffOutputFlags`, `matchCommand`. Command name = the **longest** registry entry prefixing the leading non-flag run (`source trust promote` beats `source`); everything else forwarded in **original order**. `--` = verbatim passthrough boundary. Globals stripped anywhere: `--json --plain --no-color --quiet/-q --verbose/-v --help/-h --config <path>`.
- **`handlers.ts`** — `RunContext`, `HANDLERS` map, `registerCommand`. **Deliberately a leaf module, not in `main.ts`**: command modules `registerCommand(...)` at import time and `main.ts` side-effect-imports `commands/index.js`; if `HANDLERS` lived in `main.ts` that import would hit a temporal-dead-zone crash.
- **`commands/index.ts`** — the registration barrel: **42** side-effect imports registering **52** commands (jobs/source/note/index-ops/db-* each register several). `commands/pagination.ts` = the shared pagination contract (`DEFAULT_LIMIT=50`, `MAX_LIMIT=500`, strict `INT_LEXICAL_RE`, `assertOffsetInRange`). `commands/store-open.ts` = `openMigratedStore` (read paths, never migrates) + `registerFeatureMigrations`.
- **`commands/job-handlers.ts`** — the **production workflow→executor registry** (`buildJobHandlers(deps)` + the `PRODUCTION_WORKFLOWS` union), built **per-drain** in `jobs run` because real handlers need a `RunContext` + open `Store`. Two enforced properties: **completeness** (every workflow production code can `enqueue()` is declared and registered — bound to the enqueue-side constants by `test/jobs.registry-completeness.test.ts`) and **laziness** (handlers resolve deps INSIDE the closure, so the gate can build the whole registry with a stub `deps`). Covers `retention:*`, trust remediation, `reverify`, and `index:reconcile`. The env-gated test handler (`jobs-test-handler.ts`) keeps a separate import-time seam and is spread FIRST so it can never shadow a production workflow.
- **`config/{schema,load}.ts`** — Zod **strict** schema for `brain.config.yaml`, **12 required sections** (`vault sqlite lancedb indexing retrieval git models policies jobs logs broker quarantine`). Owns every plan threshold. `ConfigError` carries `exitCode=2` + `{file, key?}`. `retrieval.rrf.weights.vector` is `.gt(0)` (a zero vector weight annihilates the §6 FTS-fallback fusion); `fts` may be 0. `load.ts` applies `ATLAS_*` env overrides then strict-parses. **`config/canonical-ref.ts`** — `DEFAULT_CANONICAL_REF` (`refs/atlas/main`) + `protectedRefsFor(cfg)` (derives the broker's protected-ref set from the config; used by `ingest/wiring.ts` + adoption bootstrap).
- **`render/safe.ts`** — `render()` is the single human-output path; `sanitize()` strips CSI/OSC (incl. OSC-8/OSC-52), lone `ESC`, C1, C0+DEL **except `\t`/`\n`** (CR IS stripped — kills CR-overwrite), and bidi controls. Colour-free by construction (`NO_COLOR`-clean). `render/plain.ts` `resolveOutputMode`; `render/progress.ts` append-only, **stderr** by default.
- **`errors/envelope.ts`** — `CliError` (single failure type) + `EXIT = {OK:0, VALIDATION:1, CONFIG:2, SECRET_SCAN:3, INTERNAL:4, USAGE:5, ACTION_REQUIRED:6}`. `emitJson` (NDJSON success), `emitError`.
- **`diag/logger.ts`** — JSONL logger, size-rotated on write (`logs.max_bytes`/`max_files`) + `sweepLogRetention` (the AGE/scheduled complement the `retention:log-rotation` job drives — age-rotates a quiescent active log the size trigger can't reach, hard-deletes rotated files past `logs.retention_days` or beyond `max_files`; matrix row 26). Redaction boundary: `msg` restricted to a stable `EVENT_ID` token (else `[redacted-message]`); `redact()` scrubs `SENSITIVE_KEY`-matching context; reserved fields (`ts/level/runId/jobId/msg`) can't be spoofed.

### Exit codes (`0 ok · 1 validation · 2 config/vault/lock · 3 secret-scan · 4 internal · 5 usage · 6 action-required`)
**No single-error-envelope command emits process exit 7.** The contracts' nominal "provider-retryable (7)" rides as exit 4/6 **carrying `retryable:true` + optional `retryAfterMs` on the envelope** — retryability is a flag, not a code (`query.ts`, `index-ops.ts`). The one exception is the `jobs run` batch aggregate: `runner.ts` `aggregateExit` maps a transient-but-exhausted item to 7 and `main()` `process.exit`s it unclamped (jobs-run schema `exitCode` enumerates 7).

## Engine

- **`workflows/`** — the crash-safe persisted run state machine (`planned→patched→worktree-applied→agent-committed→[review-pending]→integrated→reindexed→finalized`; terminals `failed/cancelled/rejected/rolled-back`), **one atomic SQLite write per transition**, driven by [`recovery-state-machine.md`](../../docs/specs/recovery-state-machine.md). `engine.ts` (`RunHandle`/`startRun`), `checkpoints.ts` (transition graph + CAS asserts), `reconciler.ts` (startup reconcile: ledger drain → run sweep → orphan sweep), `idempotency.ts` (caller-idempotency per `(command, --idempotency-key)`), plus lifecycle drivers (`approve`/`rollback`/`refresh`/`reverify`/`review`/`rebuild-from-git`), integrators (`broker-integrator.ts` is PRODUCTION; `integrate.ts` `makeSynthesisIntegrator`'s general-advance seam is **unwired**), and `ops/` projection executors. `reverify-recover.ts` (#217) = the PRODUCTION reverify seams: `recoverReverifyAnchor` re-normalizes the blob's CANONICAL-ref bytes through the real `@atlas/sources` sandbox, proves the output IS the requested rendition (version pair + recorded hash), then hash-verifies the old `char:`/`byte:` locator range in the new text against `quote_hash` — every doubt ⇒ `null` ⇒ `pending` (Tier-3); `applyReanchorViaBroker` integrates the plan through the same `applySynthesis` broker path `evidence resolve` drives. A secret hit during re-normalization rethrows PERMANENT (`validation`) — a bare `SecretDetectedError` classifies transient and burns the retry budget.
- **`policies/`** — deterministic risk (`risk.ts`, monotonic-up, proposer tier NEVER read), sensitivity (most-restrictive-on-read), `mutation-policy.ts` (byte-equal to [`workflow-risk-contract.md`](../../docs/specs/workflow-risk-contract.md)), `operation-gate.ts` (Phase-2 refuses synthesis+trust; unknown op ⇒ synthesis, fail-closed).
- **`trust/`** — fail-closed resolution, transitive taint (**floors, never averages**; empty set ⇒ untrusted), revoke (integrated ⇒ Tier-3 remediation job). The read surface (`source list`/`show`/`trust show`) reports the real `0010_trust_state` projection via `readTrustRecord` (#218) — no row ⇒ untrusted, suspended ⇒ effective untrusted; the projection holds the LATEST transition only, so `trust show` history has at most one entry (the trust ledger's Phase-1 advance is authorize-only).
- **`graduation/`** — fail-closed real-vault onboarding: `scan.ts` (working-tree + git-history), `state.ts` (`scan-state.json` sidecar gate), `audit.ts`, `migrate-plan.ts` (**PURE** two-pass id derivation), `migrate-apply.ts` (byte-exact, resumable `.checkpoint.json`, `.bootstrap-backup/` pre-images, rollback). See [`bootstrap-migration.md`](../../docs/specs/bootstrap-migration.md) (partly stale — the open-type-system superseded its refusal branches; fixtures are authoritative over prose).
- **`retrieval/`** — `layers.ts` (identity short-circuit id→slug→alias, else fenced hybrid + RRF), `rrf.ts` (`Σ weight/(k+rank)`, `k=60`, UTF-8-byte tie-break), `pack.ts` (token-bounded). Contract: [`retrieval-index-contract.md`](../../docs/specs/retrieval-index-contract.md).
- Other engine dirs: `synthesis/` (`model-output.ts` — the model-output orchestration boundary; the SSOT operation gate runs fail-closed before any sink), `markdown/` (patch engine — no whole-file rewrites by construction), `vault/` (reader errors-as-values, temp-fsync-rename writer; `note-matcher.ts` = the glob filter for `note_globs`), `validation/`, `ingest/` (scan-before-persist), `quarantine/` (AES-256-GCM ciphertext-only, key custody platform-chosen), `purge/`, `locks/` (four totally-ordered scopes ⇒ deadlock structurally impossible), `retention/`, `audit/` (read-only WORM-anchor verify), `broker/` (CLI side of challenge/authorization), `sync/` (60-A adoption: `seed.ts` = `seedSyncCursor` idempotent INSERT, `seed-cli.ts` = subprocess bridge from `adopt-vault.sh`; 60-B: `resolve-at-ref.ts` = the CLI-side `(noteId) → ParsedNote | null` resolver reading blobs at a ref, `reconcile-handler.ts` = the `index:reconcile` job handler; the sync cycle engine lands here in Phase 4).

## Invariants & guardrails

- **Single human-output channel.** Only `render/safe.ts`, `render/progress.ts`, `errors/envelope.ts` may write stdout/stderr or `console.*` — enforced by `test/no-render-bypass.test.ts` (walks all of `src/`, allowlist pinned).
- **`runCli` never throws.** Every failure serialises (JSON) or renders + maps an exit code; `main()` is the last-resort guard (`test/main.test.ts`).
- **Terminal-injection safety is release-blocking** — `sanitize()` is the trust boundary before any byte hits a TTY.
- **`implemented:true` ⇒ handler registered** (`test/command-registration.test.ts`) — guards the #145 failure class (a typo'd/dropped registration merges green, dies live as `not-implemented`).
- **`db migrate` is the sole migration composition root.** It calls `store-open.ts` `registerFeatureMigrations(store)` (jobs `0002`, workflows `0006`, generation `0008`) *before* `store.migrate()`. Every other ledger command opens an already-migrated store via `openMigratedStore` and never applies DDL (read-only reads included; `db status`/`inspect`/`status` stay available even when the backup watermark blocks writes).
- **Privileged mutations are broker-authorized, never `--yes`.** `db restore`, `purge`, `graduation migrate --apply/--rollback`, `git approve/rollback`, `source trust promote/revoke`, `quarantine resolve` follow `--export-challenge`→sign→`--authorization <path>`; no authorization ⇒ exit 6. `--yes` never authorizes (documented).
- **Preview-by-default** for `ingest`/`enrich`/`reconcile`/`maintain`/`purge` (mutate only with `--apply`; `--dry-run`+`--apply` ⇒ usage exit 5). Tier-3 review-pending ⇒ exit 6.
- **Risk deterministic + monotonic-up · trust floors · sensitivity most-restrictive · operation gate fail-closed.** The broker independently **re-derives** risk/policy/trust/evidence from the candidate tree — a compromised CLI can't launder trust. The CLI **never holds the audit-attestation key**; every canonical-ref move funnels through the broker via an injected `RunIntegrator`.
- **`ATLAS_ROOT`** overrides the auto-detected cli-contract root; precedence: `options.root` → `env.ATLAS_ROOT` → `findRoot` (walks up for `docs/specs/cli-contract/commands.json`) (`main.ts:91-95`).

## Gotchas & sharp edges

- **Egress-bearing commands need `ATLAS_EGRESS_CAPABILITY_KEY` exported** — `query`, `index rebuild`/`repair`/`eval` mint capabilities against the same secret the egress daemon verifies; a missing/mismatched key surfaces as a broker refusal. Live-drive gotcha; see the runbook below.
- **`sniffOutputFlags` mirrors `parseArgv`'s global-flag spelling** — a non-fallible second copy run *before* the fallible parse so a usage error still honours `--json`. A new global flag must land in **both** or the pre-parse-error path won't honour it.
- **`--idempotency-key` is parsed-but-unused by some commands** (`enrich`/`reconcile`/`graduation migrate` consume flag+value but don't thread it); `jobs`/`source add`/`note add`/`ingest` DO honour it via the persisted caller-idempotency layer.
- **`coerceLike` passes bad env-override values through** to let the schema reject them with a clear message (`ATLAS_INDEXING_DIMENSIONS=foo` fails as a schema error, not silent coercion). Nested keys resolve longest-known-prefix-first; array overrides split on commas.
- **Installed-binary root resolution is layout-dependent** — a packaged install without the repo layout needs `ATLAS_ROOT` set or `loadRegistry` fails `internal`. Personal-project posture: run from the repo.
- **`indexNotes` consumes a `ParsedNote` PROVIDER, never fences.** `IndexDeps.notes` is `() => ParsedNote[]` — re-embedding needs the note BODY to chunk + embed, and `indexNotes` infers a REMOVAL from a requested id being **absent** from `deps.notes()`. So `index:reconcile` wires `deps.notes` to `scopedNotesProvider(resolveAtRef(repo, config.git.canonical_ref, config.vault.note_globs), noteIds)`: a surviving id yields its parsed note, an unresolvable one is omitted ⇒ its chunks are dropped. Passing `noteFencesForNotes(...)` here (as the 60-B plan sketch does) is un-re-embeddable and wrong — fences are the `computeStaleness`/`indexVerify` input, not an `IndexDeps` field.
- **Engine `round-2`/`round-3 finding` comments are load-bearing** — CAS/ordering fixes closing real crash-safety holes from the #74 review; don't "simplify" `expectedFrom`/`assertAdvanced`/`expectChanges:1` away.
- **`RunHandle.integrate` never fabricates `canonicalSha`** — an anchored `run.integrated` event does NOT prove canonical advanced (append precedes the CAS). `recordIntegration` can record `integrated` without `WorkflowDeps.repo` (containment proof skipped) — treat `repo` as effectively required for integration paths (#5/#65 residual).
- **Graduation migrate is PURE** (no wall-clock/FS): reserve-all-explicit-ids-first, sorted-path-first owner wins, slug loser renames + alias loser drops the alias (**both** — #153 fixed "rename only, colliding alias left behind"). `refused`/`releases` arrays retained-but-always-empty since the open type system (#151).
- **Batch commands** (`jobs run|retry|cancel`) are the sole exception to the single error envelope: they emit `{command, items[], aggregate}` even on partial failure, aggregate `exitCode` derived from the batch.

## Live-drive runbook (from [`2026-07-18-search-index-live-drive-retro.md`](../../docs/retros/2026-07-18-search-index-live-drive-retro.md) — authoritative over any plan)

A full-corpus drive against a graduated vault: (1) the drive broker needs its OWN vault repo — a **fresh clone** of grad-copy (clone drops custom `refs/audit/*`; grad-copy's `refs/audit/runs` collides seq 0) + a fresh anchor; (2) commit the migration BEFORE cloning, and `git rm -r .bootstrap-backup` from the clone (else `db rebuild` reads ~209 frontmatter-less originals); (3) fresh ledger ⇒ `db migrate` before `db rebuild`; (4) export `ATLAS_EGRESS_CAPABILITY_KEY` for every mint-bearing command; (5) start the broker only AFTER the copy exists (it validates `refs/audit/runs` at startup); (6) the apply challenge nonce has a short TTL — sign + apply promptly.
