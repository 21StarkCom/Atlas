# console/ — CLAUDE.md

Atlas Console — a **SwiftUI macOS app** (SP-2 "Console Cockpit"), one Swift Package with three targets. It is a **pure read-face** over `brain watch --json` + the read-class `--json` surface, and a **privileged-flow driver** (render challenge → shell to `atlas-signer` → re-invoke `brain … --authorization`). It opens **no broker socket**, holds **no broker/signing credential**, imports **no atlas internal package**, and handles the egress capability key transiently. Preserve that boundary.

**Outside the pnpm workspace.** `console/` is deliberately not in `pnpm-workspace.yaml`'s globs (`apps/*`, `packages/*`, `tools`). It builds/tests independently: `cd console && swift build && swift test`. Never run `pnpm -r test` for console work; never add it to a pnpm glob.

- **Toolchain:** swift-tools 6.0, `platforms: [.macOS(.v15)]` (Swift 6 language mode / complete concurrency). Swift 6 on PATH.
- **CI:** `.github/workflows/console-ci.yml` (`macos-15`): `swift build` → `swift test` → `scripts/assemble-app.sh`, every step `working-directory: console`.

## Module graph (no back-edges)

`ConsoleCore ← ConsoleUI ← AtlasConsole`. `ConsoleCore` (all non-UI logic) is the leaf; `ConsoleUI` (SwiftUI views) depends only on it; `AtlasConsole` (the `@main` executable, Phase 6) depends on `ConsoleUI`. `ModuleAcyclicityTests` parses the resolved package dump and fails on any library→executable back-edge or a platform other than macOS 15.

## The .app is script-assembled, not xcodebuild'd

A SwiftPM executable has no application target/scheme that emits a launchable bundle, so `scripts/assemble-app.sh` builds `-c release`, lays out `.build/AtlasConsole.app/Contents/{MacOS,Resources}`, installs `Resources/Info.plist` (`CFBundleIdentifier=com.atlas.console`), writes `PkgInfo`, and ad-hoc codesigns. It `cd`s to its own package root first, so it runs correctly from the repo root **or** `console/`. Set `ATLAS_CONSOLE_SCRATCH` to relocate the SwiftPM build dir (tests use this to avoid lock contention with the outer `swift test`); the `.app` always lands at `console/.build/AtlasConsole.app`. That path is gitignored.

## Phase 1 — foundations (`ConsoleCore`)

The substrate everything downstream binds to. Constants live once in `Constants.swift` (`ConsoleConstants` + `BackoffPolicy`).

| File | What it owns |
|------|--------------|
| `ProcessRunner.swift` | The sole subprocess-launch owner. `SpawnRequest`/`SpawnResult`/`SystemProcessRunner`. One-shot `run` drains stdout+stderr **concurrently** (no pipe-capacity deadlock), enforces `timeout` via SIGTERM→SIGKILL reap (`SpawnError.timedOut`), honors cancellation. `executable[0]` MUST be absolute (`Foundation.Process` does no PATH expansion) — a non-absolute token is a typed error. |
| `StreamHandle.swift` | Long-lived reader for `brain watch`: **raw byte chunks** (framing is Phase 2), plus `completion()` → exit code + **full captured stderr** (never swallowed). |
| `SchemaValidator.swift` | Strict runtime JSON-Schema (draft 2020-12 subset) engine. Constraint values read from schema bytes at runtime — nothing hardcoded. Implements exactly the keyword set the atlas schemas use incl. applicators + `unevaluatedProperties` (annotation propagation). `implementedKeywords`/`ignoredKeywords`/`collectKeywords` back the coverage gate. `decode(_:from:)` is the validate-then-`Decodable` wrapper. |
| `ContractBundle.swift` | Binds `commands.json` + per-command `*.schema.json` + `watch`/`error-envelope` schemas from the **checkout that supplies the CLI entry** (`resolve(fromAnchor:)` walks up from `dist/bin.js`, never the `node` launcher). `executionClass` is read from each schema's `x-atlas-contract` (it is **not** on the registry row). |
| `BinaryResolution.swift` | Resolves `brain`/`signer` first-hit-wins (settings → env var → repo-layout default), resolves bare `node` to absolute via the request PATH, probes (`brain db status --json` — a pure command, never an audited read; `atlas-signer pubkey`) under a 10 s timeout, and **fails closed** (`BlockingResolutionError` naming the path + remediation, no fallthrough). V1 same-checkout signer restriction enforced: signer resolution **requires** the already-resolved brain's `contractAnchor` (`resolve(.signer, …, brainAnchor:)`) and derives the bundle/checkout exclusively from it, so a brain from checkout A can never pair with a signer/root from checkout B; the resolved signer's `contractAnchor` is that same brain anchor. Env vars: `ATLAS_ROOT`, `ATLAS_BRAIN_PATH`, `ATLAS_SIGNER_PATH`. |
| `SignerContractValidator.swift` | Transcribed §7.1/§7.2 challenge/response validator (no importable schema). Every required field incl. `schemaVersion`/`payloadCanonicalization`; signature prefix `p256:`/`ed25519:`; **recursively** checks the echoed response challenge equals the committed one. |

## Phase 2 — stream decode, framing, exit interpreters, observability (`ConsoleCore`)

Turns the raw-byte `StreamHandle` into typed, strictly-validated stream items and logs every spawn under an argv allowlist. Nothing here holds a credential or opens a socket — pure transform over Phase-1 primitives + the bound `ContractBundle` schemas.

| File | What it owns |
|------|--------------|
| `WatchEvent.swift` | The 8 event payload structs (each carries the common envelope `at` timestamp — Phase 3 needs `snapshotAsOf = hello.at`) + `WatchEvent` (8 cases + `.unknown(raw:)`) and `WatchEventDecoder`. **Envelope-first decode order:** parse JSON → validate the common envelope (`v`/`event`/`at`, a subschema built from `watch.schema.json`'s shared `$defs`) → **unknown-event check BEFORE the closed union** (a future additive `event` ⇒ `.unknown`, never a decode error) → for a known `event`, validate the full line against that union member (`{"$ref":"#/$defs/<member>","$defs":…}`) then typed-decode. `null` where the contract omits fails at the schema gate. |
| `NDJSONFramer.swift` | Byte-level line framer. Buffers partial lines across chunks and splits on the `\n` byte only, so a multi-byte UTF-8 scalar split across a chunk boundary is reconstructed for free (no decode until a full line is assembled). `push`/`finish`; empty chunks tolerated. |
| `ExitInterpreters.swift` | Two DISJOINT enums — `BrainExit` (0–6) + `SignerExit` (0–5) — with `interpret(_:)` mapping the batch-only `7` to `.aggregateRetryExhausted` and unknowns to `.unrecognized`. Distinct types make a cross-table lookup a compile error. |
| `ErrorEnvelope.swift` | `ErrorEnvelope` (custom `Decodable`: top-level required fields, nested `errors[]` default hint/retryable per the `nestedError` shape) + `ErrorEnvelopeParser` (validate against `error-envelope.schema.json`, then decode). Retry rides `retryable`/`retryAfterMs`; `details` read by field, never by parsing `message`. |
| `StreamTransport.swift` | `StreamItem` (`.event` \| `.terminalEnvelope`) + `WatchStreamParser` (classifies a framed line by the presence of the `event` key; a blank/whitespace-only line is REJECTED — `StreamParseError.blankLine` — never silently skipped) + `WatchTransport` — the **production adapter** wiring `StreamHandle.bytes` → `NDJSONFramer` → `WatchStreamParser` into an `AsyncThrowingStream<StreamItem>` (the one framing pipeline `WatchSupervisor` consumes; `TransportFramingTests` proves four adversarial deliveries yield identical sequences). |
| `ConsoleLog.swift` | `ArgvClassifier` (schema-driven argv allowlist: binary/command/**schema-declared** flag names + structural operands intact; a free-text operand → `<redacted:<cmd> len=NN>` where `NN` is the UTF-8 **byte** count, and a free-text operand spanning **multiple bare tokens** (the NL query) is redacted in full — once a sensitive positional is reached, every following positional is redacted too; a value-flag's VALUE is logged intact ONLY if the schema affirmatively classifies it as structural (numeric-in-`constraint`-range) or enumerated — otherwise redacted (`--k=<redacted:val len=NN>`), so a misplaced secret never lands in a log; sensitive operands keyed by **schema arg name**, position resolved from the schema at runtime; **fails closed** — a malformed/unresolved schema redacts every positional, a dash-prefixed or slash-containing operand is redacted not passed through, `--` ends flag parsing, no schema ⇒ redact all operands) + `ConsoleLog` (`os.Logger`, subsystem `com.atlas.console`; `spawn`/`failure` + testable `*Line` builders; raw child stderr is recorded by byte length only via `failure(…, rawOutput:)`, never verbatim — surfaced to error surfaces, never persisted). |
| `LoggingProcessRunner.swift` | The enforcement seam: a **command-agnostic** `ProcessRunner` **decorator** wrapping any inner runner, owning every spawn/termination/failure log call. Command + schema ride each `SpawnRequest` (`command`/`commandSchema`), so ONE wrapped runner sanitizes every command via `ArgvClassifier` — nothing bound at wrap time. `run` emits one launch record (with exit code); `stream` emits one launch record (exit nil) **plus** exactly one `recordTermination` (exit code + byte-length-only stderr) when the streamed process exits. `SpawnLogging` sink protocol + `ConsoleLogSink` (production) make it test-observable. |
| `ProcessRunnerComposition.swift` | The composition root: builds the one shared `SystemProcessRunner`, wraps it once in `LoggingProcessRunner`, and exposes ONLY `runner: ProcessRunner` (never the concrete inner). No API path to an unwrapped runner ⇒ no component can bypass logging (asserted by `testCompositionRootRunnerCannotBypassLogging`). |

## Phase 3 — reducers (`ConsoleCore`)

Pure, fixture-driven transforms over decoded `WatchEvent` payloads (Phase 2). No sockets, no credentials, no atlas-internal imports — a run-space head, a snapshot+live-overlay dashboard, a per-job count map, and resume/cursor selection. Load-bearing invariant across all four: the **disjoint seq spaces** (`run.*` seqs `< DB_EVENT_SEQ_BASE = 10^12` vs the high-space `db.*` + `evidence.retry_enqueued` live-only range).

| File | What it owns |
|------|--------------|
| `AuditReducer.swift` | Run-space-only audit timeline. `apply` accepts an `audit` event into the seq-ordered timeline **iff `seq < DB_EVENT_SEQ_BASE`**; a high-space event routes to `.highSpaceSignal(kind)` (kind classified from `eventType`, unknown kinds surfaced not dropped) and **never** touches the timeline, head, or cursor. `incorporateHello(baselinePrefix:)` (a full re-baseline, called on every hello) seeds the contiguous-prefix head — `-1` = replayAll/fresh/detached, a reported `hello.resume.auditHeadSeq` = existing-ledger baseline (avoids the phantom-gap freeze). `displayedAuditHead == safeCheckpointSeq == max contiguous run-space seq`; a gap is a pending intent (never pruned), so neither advances past it. |
| `DashboardReducer.swift` | `hello.snapshot` overlaid by live events, **one derivation per field**: live-updated (`backup{watermarkSeq,healthy}`, `audit.headSeq` from an embedded `AuditReducer`, daemon reachability, `openRuns` via run-lifecycle deltas) applied only **after** `markCheckpointReached()`; snapshot-only (`quarantineCount`, `backup.coveredSeq`, `audit.anchorOk/anchorSource`) held at the last hello, labelled "as of `snapshotAsOf` = hello.at", **never fabricated as `0`** (absent ⇒ `nil`). Replay-phase run-state deltas are inert (already in `snapshot.openRuns`), so a replayed `run.started` never double-counts. `DashboardState` + `BackupView`/`AuditView`/`DaemonView`. |
| `JobState.swift` | The snapshot-plus-delta job counter. `JobStateMap` recency-merges by `updatedAt` (a list row never clobbers a newer streamed state) and recomputes `JobCounts{queued,failed}` (queued = pending/ready/running, per `health/snapshot.ts`). `JobsListReader.readAll(invoker:)` fully consumes `jobs list --json --limit 500` pagination through the `ReadInvoker` seam (`run(_:args:) -> Data`; phase 6 passes `ReadCommandExecutor.run`, so no raw runner in the reader) — walks `offset` to `hasMore:false`, **dedups by `jobId`**, refuses a non-advancing page. `JobStateCoordinator` (actor) is the **sole** seed/reseed driver + view surface: it owns `pendingBuffer`/`reseedInFlight` so a single read is ever in flight and every event arriving during the read is merged; an unseen `jobId` ⇒ buffer the live event, `readAll`, recency-merge onto **list-confirmed membership only** (never a synthetic insert). Exposes `rows: [JobListRowState]` (list order = `createdAt`-desc/`jobId`-tiebreak, live state overlaid) and `refresh()` (explicit read-on-focus full re-read). |
| `ResumePlanner.swift` | `ResumeMode`/`ResumeArg`. `plan(mode:persistedCursor:)` — `resume` → cursor `>= 0` ⇒ `.sinceSeq(cursor)`, else `.liveOnly`; `replayAll` → `.sinceSeq(-1)`; `liveOnly` → `.liveOnly`. `isStaleCursor(replayEvents:resumeHead:requested:)` = `events==0 && resumeHead < requested` (cursor-above-head only; catch-up-past-cursor is the accepted V1 residual — escape `replayAll`). |

## Phase 4 — watch supervisor & local stores (`ConsoleCore`)

The live-attach control plane: the durable resume cursor, the settings + schema-derived option bounds,
the gated-backoff supervisor, daemon/detach transitions, and the coordinator that threads them into a
checkpointed live stream. Still a pure read-face — no broker socket, no credential, no atlas-internal
import.

| File | What it owns |
|------|--------------|
| `CursorStore.swift` | `CursorStoring` seam (so a test can inject load/checkpoint failures) + the single-writer resume-cursor SQLite store at `~/Library/Application Support/com.atlas.console/console.sqlite` (parent dir `0700`, db `0600`, one `ledger_cursor(incarnation_key PK, audit_head_seq DEFAULT -1, updated_at)` table). `IncarnationKey.derive(ledgerPath:)` = SHA-256 of the absolute path — a re-cloned/re-anchored vault at the same path is a fresh key (fresh `-1`, never a stale cursor). All access serialized behind a lock in a connection holder; `checkpoint` is transactional (`BEGIN IMMEDIATE`/`COMMIT`/`ROLLBACK`). Exposes `checkpointCount` for the sole-writer invariant. `load` returns `-1` for an unseen key. |
| `Settings.swift` | `Settings: Codable` blob (all overrides optional) + `static let defaults` (every optional nil, `.env`, `.resume`). `SettingsStore` over a `UserDefaults` suite distinguishes an **absent** blob (⇒ pure defaults) from a **corrupt** one (⇒ defaults + `.resetFromCorrupt` notice, never a crash/invented value) — presence is checked via `object(forKey:)` FIRST (so a present-but-wrong-type value reads as corrupt, not absent, since `data(forKey:)` returns nil for both). `resolutionInputs()` maps into Phase-1's `ResolutionInputs`. `WatchOptionPolicy` derives `--poll-ms`/`--heartbeat-seconds` ranges + defaults from the bound `watch.schema.json` `x-atlas-contract.flags` `constraint`/`default` (SSOT-derived, fails closed on a missing flag table) — an absent override OMITS the flag so the CLI owns its default, and `watchOptions(from:)` **validates** every persisted override at the settings→argv boundary (a Codable-from-`UserDefaults` out-of-range value is DROPPED, not passed through — proven against both the current and a mutated fixture range). |
| `WatchSupervisor.swift` | `actor WatchSupervisor` — the sole periodically-polling subprocess owner. `classify(exit:envelope:)` is **total** over `(exitCode, envelope?)`: exit 0 ⇒ cleanDetach; exit 5/2 ⇒ terminal (structural); other nonzero decided by the envelope (retryable flag), on-contract code + no envelope ⇒ retryable (dropped stream), **off-contract code + no envelope ⇒ terminal fail-fast**. A framing/decode failure while the child is alive ⇒ terminate + await child ⇒ terminal `contractMismatch` (never hangs on `completion()`). Gated backoff (`BackoffPolicy.delayMs` — 500ms ×2, 30s cap, ±20% jitter, `retryAfterMs` floor) + the consecutive-failure counter reset **only on a proven-healthy run** (a `hello` followed by its first attached `heartbeat`); `WATCH_MAX_CONSECUTIVE_FAILURES = 6` (initial + 5 respawns). Publishes a `stateChanges` `AsyncStream<SupervisorState>` on every transition (`.retrying` carries the ACTUAL failure identity — the envelope `code` / `exit-<n>`, never a literal "retryable"). Terminal states carry the transient diagnostics the contract requires: `.failed(exit,code,hint,stderr)` keeps the envelope `hint` (nil when absent, never fabricated) AND the child's full captured stderr; `.contractMismatch(stage,stderr)` keeps stderr too — neither is discarded. The schema-**unbounded** `retryAfterMs` floor is applied in **integer** space (never `Double(Int.max)`, which would trap on the round-trip) and the `nextAt` banner uses **saturating** `addingReportingOverflow` (a huge floor saturates to `Int.max`, never overflows). The backoff sleep is **wakeable**: `stop()` interrupts an in-flight backoff immediately (never a 30s / unbounded-`retryAfterMs` stall) and reaps the child with a **bounded SIGTERM→SIGKILL escalation** (`reapGrace`, injected short in tests), so a TERM-resistant child never hangs the contract-mismatch reap or `stop()`. `events` stream stays open across `stop()`/re-`run()` so a coordinator can re-attach a new incarnation. |
| `TransitionRouter.swift` | `struct TransitionRouter` turns `WatchEvent`s into `[UISignal]` for the daemon/detach/degraded surfaces. `ReachState.notInstalled` is DERIVED (unreachable AND `socketPath` absent on a read-only stat — never opens a socket; present-but-refusing ⇒ `unreachable`). Emits reachability changes, attach/detach, `reattaching` on `watch.error(source:"ledger")` (which INVALIDATES the cached attach state so the awaited same-ledger `attached` hello re-emits `.attached`, clearing `reattaching`), `anchorDegraded` on `anchorSource:"sqlite-only"`, `backupHealth` (never blocks the stream), and the distinct `serviceNotInstalled` empty state. |
| `AttachCoordinator.swift` | `actor AttachCoordinator` — the SOLE owner of once-hello → cursor → plan → live-spawn → checkpoint. Spawns `watch --json --once` (via `runner.run`), derives the `IncarnationKey` (detached ⇒ skip cursor, plan live-only), loads the cursor, plans the `ResumeArg`, and runs the supervisor. Consumes the supervisor's events, forwarding them, and **checkpoints only on safe attached heartbeats carrying `resume.auditHeadSeq`** (never a pre-replay hello value, never a detached heartbeat) — the sole checkpoint writer. A same-incarnation re-`hello` (startup / post-retry) rebaselines in place; only a real incarnation transition (detached↔attached flip or changed `ledger.path`) calls `supervisor.stop()`, awaits both the process exit and the run-loop task, then re-plans for the new incarnation. Every event is **generation-guarded BEFORE forwarding** (not only before checkpointing): a heartbeat belongs to the active generation iff its own `ledger.path` derives the active incarnation key (attached) or the coordinator is live-only (detached) — an invalidated-generation heartbeat buffered behind a transition hello reaches neither the reducers nor the cursor store. Storage failures are **surfaced, never swallowed**: a `CoordinatorState`/`stateChanges` stream carries `.attaching`/`.live`/`.degraded`(checkpoint failed, non-terminal)/`.failed`(cursor-load or replacement-generation failure, terminal) — a startup cursor-load failure throws out of `start()` and rolls the lifecycle back (`started` stays false, the parked consumer reused) so a retry succeeds, but a **replacement-generation** cursor-load failure **latches** a one-way terminal state (`terminated`): consumption stops, no further generation ever spawns, and a later buffered hello / a re-`start()` (`AttachError.terminated`) can never quietly resurrect a failed coordinator. |

## Tests

`cd console && swift test`. All Phase-1 suites run in CI (no Secure Enclave, no real broker — software fixtures + a scripted-spawn harness). `TestSupport` locates the repo checkout from `#filePath` and builds throwaway fixture checkouts. Named Phase-1 suites: `ModuleAcyclicityTests`, `AppBundleIdentityTests`, `ProcessRunnerTests`, `StreamHandleTests`, `SchemaValidatorTests`, `SchemaKeywordCoverageTests`, `SchemaDecodeTests`, `ContractBundleResolutionTests`, `PathResolutionProbeTests`, `NodeResolutionTests`, `SignerContractValidatorTests`. Named Phase-2 suites: `WatchEventDecoderTests`, `UnknownEventToleranceTests`, `NDJSONFramerTests`, `ExitInterpreterTests`, `ErrorEnvelopeTests`, `TransportFramingTests` (real subprocess-pipe framing under adversarial chunking), `ArgvSanitizationTests`. Named Phase-3 suites: `MixedSpaceReducerTests`, `LiveOnlyExistingLedgerBaselineTests`, `DashboardReducerTests`, `CoalescingJobCountTests`, `ResumeReplayTests`, `StaleCursorRebaselineTests`, `RebaselineOnHelloTests` (fixtures + a scripted `jobs list` `ReadInvoker` — `ScriptedReadInvoker` in `ReducerFixtures.swift`, with an `onPage` pause hook for the paused-paginator concurrency test, plus `GatedReadInvoker` (a read that parks until `release()`) for the concurrent-seed/refresh in-flight-await + failure tests — payloads built directly since decoding is proven in `WatchEventDecoderTests`). Named Phase-4 suites: `CursorStoreTests`, `LiveOnlyCheckpointSurvivesRestartTests`, `SettingsStoreTests`, `WatchSupervisorTests`, `DaemonTransitionTests`, `AttachResumeCheckpointFlowTests` — the supervisor + coordinator suites drive a `ScriptedSpawnRunner` (`WatchTestHarness.swift`) that pops per-`stream()` behaviors and launches real emitter shell scripts (real pipes/exit codes, like `TransportFramingTests`) while returning canned `--once` stdout with no subprocess; `Fx4` builds schema-valid NDJSON lines from the `watch.schema.json` examples. The supervisor's backoff `sleeper` is injected so backoff is asserted deterministically without real waits, and `CursorStore.checkpointCount` proves the exactly-one-checkpoint-at-the-heartbeat invariant. `WatchSupervisorTests` also asserts the `stateChanges` stream (retry attempt/nextAt/real lastCode), a prompt `stop()` during `.retrying` (blocking sleeper, no respawn), and a TERM-resistant child reaped via the bounded SIGTERM→SIGKILL escalation (`ScriptedSpawnRunner.emitThenIgnoreTerm`, short `reapGrace`). `AttachResumeCheckpointFlowTests` covers the generation-guard drop of a stale pre-transition heartbeat and injected cursor-load/checkpoint failures (`ToggleFailStore`) surfacing terminal/degraded states + a start-rollback-then-successful-retry. The flow tests poll generously (30 s) because they launch real emitter subprocesses under the suite's parallel load.
