# console/signer/ — CLAUDE.md

`atlas-signer` — the standalone **Secure-Enclave P-256 authorization signer** (SP-3, ADR-0002). The **one** component that ever touches the SE approver key: the terminal privileged flow uses it directly, the SP-2 Console shells out to it, and neither `brain` nor the Console ever links the key material. It holds no network, opens no socket, and never invokes `brain` — pure `stdin → display → SE → stdout`.

**A SEPARATE SwiftPM package from `console/`** (its own `swift build`/`swift test`), and **outside the pnpm workspace**. Built ad-hoc from source: `swift build` under Command Line Tools — **no Xcode, no `.app` bundle, no entitlements, no provisioning profile** (the CryptoKit `dataRepresentation` blob custody avoids the data-protection keychain entirely, so the linker's automatic ad-hoc signature suffices). **CI does not build it** (spec §6/§11) — GitHub's macOS runners are Virtualization.framework VMs with no Secure Enclave, so the SE path can't execute there; the p256 **verify** path is CI-covered on the TypeScript side (`@atlas/broker` software fixtures). An optional compile-only job rides SP-2's Swift CI (spec R5).

## Module graph

`SignerCore` (testable leaf — all non-OS logic + the protocol-abstracted signing backend) ← `atlas-signer` (thin `@main` executable: real SE backend + real IO, maps `SignExit` → `exit()`). Mirrors the Console's `ConsoleCore ← …` shape so the crypto/parse/render logic is unit-testable with a **software P-256 backend** substituted for the enclave.

## Subcommands (spec §6) — `atlas-signer <keygen|pubkey|sign>`

- **`keygen [--signer-id <id>] [--force]`** — mints a `SecureEnclave.P256.Signing.PrivateKey` under `.privateKeyUsage | .biometryCurrentSet` (Touch ID; **V1's only access policy**), persists the SE-wrapped blob + `config.json`, prints the SPKI PEM (stdout) + the `signerId` and enrollment runbook (stderr). Id convention `approver-se-<hostname>-vN`; an explicit `--signer-id` **must** carry a `-vN` suffix (rejected otherwise) so `--force` rotation always derives `-v(N+1)` from the prior `config.json`. Refuses to overwrite an existing blob without `--force`.
- **`pubkey [--out <path>]`** — writes the SPKI PEM **alone** to stdout (or `--out`), `signerId`+`accessPolicy` to stderr, so a redirect yields a valid PEM-only file for `enroll-signer.sh --pubkey`.
- **`sign [--out <path>]`** — reads one `AuthorizationChallenge` on stdin, and in this order: decode+shape-check → **re-derive `signingPayload` and refuse (exit 2) on mismatch** → check `expiresAt` (exit 3 if expired, **before** any prompt) → render the control-safe approval summary (stderr) → pre-arm one `LAContext` (summary as `localizedReason`) → sign the UTF-8 `signingPayload` bytes (`signature(for:).derRepresentation`) → invalidate the context → emit the `AuthorizationResponse`.

## Load-bearing invariants — do not break

- **Byte-identity with the broker.** `SigningPayload.rederive` is a faithful port of the broker's `buildSigningPayload`/`effectCommitmentLines` for **all nine** `intendedEffect` kinds. Pinned by the committed cross-impl golden vectors `Tests/SignerCoreTests/Fixtures/signing-payload-vectors.json`, **generated by the broker itself** (`gen-fixtures.mjs`, run against `@atlas/broker`'s `buildSigningPayload`) so drift in *either* implementation breaks `testGoldenVectorsRederiveByteIdentical`. Re-run `node console/signer/gen-fixtures.mjs` after any `buildSigningPayload` change.
- **Re-derive-and-refuse.** `sign` re-derives `signingPayload` from the challenge's own fields and refuses (exit 2) if it disagrees with the challenge's `signingPayload` — the bytes signed are provably the bytes shown. Second, independent backstop to the broker's recompute-and-compare.
- **Verbatim challenge echo.** The response embeds the exact received challenge bytes (never a re-encode), so the broker's recompute and the Console's `SignerContractValidator` echo-equality always pass.
- **base64url, not base64.** The `p256:` signature body is unpadded base64url (`Base64URL.encode`) — CryptoKit/Foundation emit standard base64, which the broker's `Buffer.from(x,"base64url")` would mis-decode. DER↔DER, SPKI-PEM keys, zero transformation otherwise.
- **Channel contract (SP-2 parses this).** Summary + every diagnostic → **stderr**. Response: with no `--out`, the JSON is the **only** thing on stdout (empty on failure); with `--out`, JSON → that file (`0600`, refuse existing without `--force`) and **stdout stays empty on success AND failure**.
- **Exit codes are ITS OWN table** (`SignExit`), matching `ConsoleCore/ExitInterpreters.swift` `SignerExit`, **not** the `brain` EXIT map: `0` signed · `1` internal · `2` malformed/invalid/re-derive-mismatch · `3` expired · `4` cancelled/biometry-failed · `5` key invalidated by re-enrollment.
- **Control-safe rendering.** Every challenge-derived string in the summary passes through `ControlSafe.render` (tokenizes Cc/Cf/Zl/Zp + pinned U+180E as `<U+XXXX>`, never truncates) so a hostile field value cannot rewrite the summary.
- **Key custody.** Blob + `config.json` at `~/Library/Application Support/atlas-signer/` (dir `0700`, files `0600`), in the **operator's** home — the agent UID cannot read it, and the blob is inert off this Mac's enclave anyway. `KeyStore`'s base dir is injectable so tests use a throwaway location.

## Build & test

```bash
cd console/signer && swift build && swift test    # SignerCore: 15 tests, software P-256 backend
./install.sh [approver.pem]                        # operator-space half of the fresh-Mac story (§7)
```

`install.sh` builds `-c release`, links `atlas-signer` into `~/bin`, runs `keygen` (fires Touch ID), `pubkey --out approver.pem`, and prints the exact `sudo provisioning/enroll-signer.sh …` line — the **separate, sudo-gated, human-typed** enrollment step that crosses into broker custody. Two operator commands total.

## Consumers & contract

- **SP-2 Console** (`console/Sources/ConsoleCore/`) shells out to `sign` and branches on its exit codes; `SignerContractValidator` validates the response shape. The #272 test re-anchoring points those fixtures at this package's golden output.
- **Broker** (`@atlas/broker`) verifies the emitted `p256:` signature via `verifyP256Bytes` after resolving the enrolled signer's `alg` (§9.2). **No fixture signer — of any algorithm — authorizes outside `ATLAS_TEST_MODE` (D20).**

## Gotchas

- **The real enclave never runs in CI or in unit tests** — `SecureEnclaveSigningBackend` is exercised only on the operator's provisioned Mac (the P6 live drive). Tests inject `SoftwareSigningBackend`. Keep every SE-specific branch behind the protocol so the leaf stays testable.
- **Exit 5 (biometry re-enrollment) is not synthesizable in CI** — it's asserted via an injected `InvalidatedBackend` in tests; the live runbook exercises only the `--force` rotation *recovery*.
- **`config.json` is JSON with `sortedKeys`+`prettyPrinted`** — a stable on-disk form; `pubkey` echoes the recorded `accessPolicy` (fixed `biometryCurrentSet` in V1).
