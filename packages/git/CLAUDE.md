# `@atlas/git` — typed git plumbing over the vault repo

Typed git plumbing over the vault repository. In **Atlas v2** (single process, [ADR-0003](../../docs/adr/0003-retire-security-architecture.md)) **git is the only safety mechanism**: every applied ChangePlan is exactly one commit on `refs/heads/main`, and that commit is both the audit trail (`git log`/`git blame`) and the undo (`git revert <sha>` + `brain sync`). This package is how `brain` **makes and reads** those commits — nothing else touches the repo.

The canonical ref **IS `refs/heads/main`** — no config indirection, no separate protected branch. There is no broker, no daemon, no second OS identity: the one `brain` process opens the vault, commits, and exits.

Zero external git library: everything shells the system `git` binary via `node:child_process` (`execFile` — argv array, never a shell string), per the task contract.

## Fit in the monorepo

- **Single dependency:** `@atlas/contracts` (workspace) — `ULID_RE`, `RunManifest`/`RunManifestSchema`, `canonicalSerialize` (`atlas-jcs-v1`). See `package.json`.
- **Sole dependent:** `apps/cli` — the vault-write substrate for every mutating command (via `commitPaths`) plus the read/diff surface `status`, `sync`, and the startup reconciler consume.

## Invariants & guardrails — do not weaken

- **`commitPaths` is a no-op on failure (for the paths it touched).** A snapshot of the touched paths' index entries is taken before staging; on a commit failure the union of pre- and post-stage matches is restored in ONE `update-index -z --index-info` invocation (a pre-stage entry rewritten to its prior mode+oid — resurrecting a staged deletion; a newly-staged path force-removed via the mode-`000000` form). One stdin-fed subprocess, so a mid-restore crash can't strand a half-restored index. If the rollback itself fails, BOTH errors surface via `AggregateError` (`commit-paths.ts:246-256`). **Unmerged paths are refused before any index mutation** — the primitive is not designed to run mid-merge (`commit-paths.ts:189-196`).
- **Deterministic authorship.** Both `commitPaths` and `Worktree.commit` stamp `Aryeh Stark <aryeh@21stark.com>` (author AND committer) with `commit.gpgsign=false`, set per-invocation via `-c` **and** the environment — `-c user.name/email` is overridden by ambient `GIT_AUTHOR_*`/`GIT_COMMITTER_*`, so a poisoned environment could otherwise rewrite the committed identity (`commit-paths.ts:64-74,207-234`, `worktree.ts:10-21,43-75`). The v2 commit lands directly on `refs/heads/main`, so its identity must be fully deterministic on its own.
- **No-protected-write source audit (still enforced).** Every ref-*writing* path routes through a guard called FIRST: `updateAgentRef`/`deleteAgentRef`/`attachHeadToAgentRef` call `assertAgentRef` (rejects anything not matching `refs/agent/<ulid>`); `advanceCanonicalRef` calls `assertCanonicalRef` (rejects the agent, `refs/audit/*`, and `refs/trust/*` namespaces + path-traversal/whitespace). `git.no-protected-write.test.ts` enforces this three ways:
  1. **Structural source audit** — `auditSources()` walks `src/` and fails if any ref-write subcommand (`update-ref`, `symbolic-ref`, `branch`, `push`, `tag`, `fast-import`) appears outside `refs.ts`, inside a `refs.ts` function lacking its guard, or as a top-level literal. `REQUIRED_GUARD` binds each writer to its **specific** guard (`advanceCanonicalRef`→`assertCanonicalRef`; the three agent writers→`assertAgentRef`) — a swapped guard fails the audit, so a writer can't be smuggled into the wrong namespace.
  2. **Public-surface** — the barrel must not re-export `runGit`/`runGitBuffer`; raw argv execution would bypass every guard.
  3. **Behavioral** — `assertAgentRef` rejects `refs/heads/*`, `refs/tags/*`, `refs/remotes/*`, `HEAD`, bare/short names, `refs/agent/../heads/main`, non-ULID, a 25-char ULID; accepts only a well-formed 26-char `refs/agent/<ulid>`.
- **Canonical advance is fast-forward-only CAS.** `advanceCanonicalRef` reads + compares the tip to `expectedOld` up front (mismatch ⇒ typed `broker.cas_failed`), asserts `newCommit` contains the current tip (`merge-base --is-ancestor`, else `broker.not_fast_forward`), and re-asserts `expectedOld` atomically on the `update-ref <ref> <new> <old>` write; only git's own old-value-mismatch wording (exit 128) is laundered to `cas_failed`, every other failure propagates (`refs.ts:180-237`).
- **ULID-pinned ref names.** `agentRef` / `AGENT_REF_RE` require the runId segment to be a ULID, so a caller cannot smuggle path segments into a ref name (`refs.ts:21-26,47-52`).
- **`null` means only "unresolved".** `readRef`/`commitTree`/`isAncestor`/`readBlobAt` translate git's specific "does not resolve" signal to `null`/`false`; every other failure (not-a-repo, corruption, missing binary) propagates as `GitError` (`refs.ts:59-75`, `repo.ts:322-365`).
- **Fail-closed name-status normalization.** The sync-cycle parsers fold git's raw alphabet deterministically (`R<n>`→`R`+`fromPath`, `C<n>`→`A`, `T`→`M`) and THROW on anything else (unmerged `U`, `X`, broken `B`) rather than silently drop a change (`repo.ts:90-133`). The RAW variant (`changedStatusesInRange`) keeps letters verbatim and reports both rename sides — the shape a whole-range scope gate needs.
- **Codec round-trip + trailer-injection resistance.** `parseManifestTrailer(buildCommitMessage(msg, m))` deep-equals `m`. `buildCommitMessage` refuses a caller message containing `Atlas-Run-Manifest:` (case-insensitive, any line); `parseManifestTrailer` rejects zero or >1 trailers rather than guessing (`commit.ts:39-46,54-69`).

## Gotchas & sharp edges

- **`headRef` reads via `rev-parse --symbolic-full-name HEAD`, NOT `git symbolic-ref`.** The no-protected-write source audit treats `symbolic-ref` as a ref-write subcommand and forbids it outside `refs.ts`; `headRef` is a pure read, so it uses `rev-parse` (git prints the literal `HEAD` for a detached head → `null`) (`repo.ts:428-435`). The mutation-order wrapper asserts this equals `refs/heads/main` before any mutation.
- **`readBlobAt` byte-exactness.** It funnels through `runGitBuffer` because `runGit` trims stdout and forces utf8 — a trailing newline IS blob content and non-utf8 bytes must survive. Binds to COMMITTED state: a working-tree-only file does not resolve (`null`) (`repo.ts:349-365`).
- **`worktreeStatus` consumes a rename's origin path.** A porcelain `R`/`C` entry is followed by its origin path as the NEXT NUL record — the parser skips it so it isn't misread as a second entry (`repo.ts:437-458`).
- **`removeWorktree` uses `--force`.** Uncommitted/locked worktree state is not a blocker to teardown; callers wanting dirty-tree protection won't get it (`repo.ts:497-499`).
- **Manifest ≠ signature.** This package embeds the manifest trailer only; there is no commit signing here (`commit.ts:11-14`).
- **Legacy/unwired seams — retained + tested, no live caller.** The agent-worktree commit flow (`createAgentBranch` → `addWorktree` → `Worktree.commit` with a manifest trailer) and the FF-only canonical integrator (`advanceCanonicalRef` / `makeCanonicalIntegrator` in `apps/cli/src/workflows/direct-integrator.ts`) survive from Phase 2/the in-process cutover but are **off the v2 mutation path** — the whole install is `commitPaths` onto `refs/heads/main`. Don't wire them back without a decision; don't delete them without pruning their tests.

## How `apps/cli` consumes it (map, don't restate)

- **Mutation install** (`workflows/mutation-order.ts` `runMutation`): the whole install is `commitPaths(repo, touchedPaths, message)` — one direct commit onto `refs/heads/main`. `headRef()` asserts `HEAD == refs/heads/main` first (feature-branch/detached ⇒ exit 2, no mutation); `worktreeStatus(paths)` is the dirty-vault grounding gate. Driven by every mutating command — `note add`, `ingest`, `link`, `enrich`, `maintain`, `evidence resolve`, and synthesis apply — through `runMutation`.
- **`status`** (`commands/status.ts`): read-only `openRepo` + `readRef("HEAD")` + `worktreeStatus([])` for the `git-healthy` / working-tree-dirty check.
- **`sync`** (`sync/diff.ts`, `sync/resolve-at-ref.ts`): `readRef` + `isAncestor` (cursor↔head relation + first-parent anchoring), `commitsInRange` (first-parent absorb walk), `readBlobAt` (byte-exact note bodies at the ref), `changedStatusesInRange` (all-reachable raw-status scope gate).
- **startup reconcile** (`workflows/reconciler.ts`, `workflows/engine.ts`): `commitTree` + `isAncestor` prove a recorded commit's tree and containment during crash recovery; `listWorktrees` + `removeWorktree` sweep orphan worktrees.
