# `@atlas/models` — the in-process typed model client

**Purpose.** The typed client for the three non-mutating provider ops — `generateText` / `generateObject<T>` / `embed` — plus CLI-side `model_calls` persistence (D6/D18). Post the **Phase-2 in-process cutover** the Gemini adapter, the provider credential, and the outbound network all live **HERE**, in-process: there is **no egress daemon, no capability mint, no per-run byte/token/cost budget, and no egress scan gate**. The only surviving control on the provider path is the per-call `maxTokens` on the request. The credential is resolved **LAZILY** on the first provider call — never at import/process start — so a non-provider command on a blank host (no `security` binary, no env var) never resolves a key or throws (`src/client.ts`, `src/index.ts`). **v2 (#338)** also retired the §2.8 audit ledger: a `model_calls` row is now a **plain operational row** — no `finalizeLedgerWrite`, no audit event, no per-run receipt journal.

Normative interface: **this doc + `src/client.ts`** (the in-process invoke surface — `ModelsClient`, `createInProcessInvoker`, `EgressInvokeParams`, `RunBinding`). The old [`docs/specs/provider-interface.md`](../../docs/specs/provider-interface.md) is **SUPERSEDED** by the Phase-2 cutover — its IPC seam, run-bound capability, in-broker scan, and `(req, cap, signal?)` signature are all gone; keep it only for the historical request/result-shape + taxonomy reference. Response-scan history (now moot): [`docs/adr/0001-egress-response-scan-released-bytes.md`](../../docs/adr/0001-egress-response-scan-released-bytes.md), **superseded by** [`docs/adr/0003-retire-security-architecture.md`](../../docs/adr/0003-retire-security-architecture.md). Root constitution: [`../../CLAUDE.md`](../../CLAUDE.md).

## How it fits

- **Depends on** `@atlas/contracts` (`SCHEMA_REGISTRY`, `Sha256Digest`, `Ulid`), `@atlas/sqlite-store` (the `LedgerStatement` type — the CLI folds `buildModelCallStatement` output into a plain `applyLedgerWrite` transaction), `zod`. Dev-only: `@atlas/testing`. **No broker dependency** — the packages `@atlas/broker` + `@atlas/scan` are **DELETED** in v2; the cutover moved the adapter + its pure deps here so nothing dangles. No runtime file in `src/` imports either.
- **Consumed by** `apps/cli` (`query`/`enrich`/`maintain`/`index rebuild`/`index eval` drive the client over the in-process invoker; `status`'s `provider-key-present` check calls the non-throwing `hasGeminiApiKey`) — no capability key, no daemon.
- `package.json`: `private`, `version: 0.0.0` — no semver (playground posture). Build `tsc -p tsconfig.json`; test `vitest run --passWithNoTests`.

## Public surface (essentials)

- `ModelsClient(invoker, receiptSink, { schemaRegistry? })` — `receiptSink` is **MANDATORY** (D6/D18). Every op's 3rd arg is `SignalOrOptions = AbortSignal | CallOptions`; a **bare `AbortSignal` MUST work** (`toCallOptions` duck-types `aborted` + `addEventListener`, never `instanceof` — cross-realm safe). Every op's **2nd arg is a `RunBinding` (`{ runId }`)** — the receipt attributes to it; there is no capability arg.
- `createInProcessInvoker(cfg?)` → `Invoker`. `cfg`: `adapter?` (tests inject a stubbed-transport adapter), `schemaRegistry?`, `resolveApiKey?`, `transport?`, `env?`. The adapter (and thus the credential) is built **lazily on the FIRST call** — never at factory creation. **`env` is the RunContext env** the fake-provider gate and the default key resolver read — every CLI composition root passes `{ env: ctx.env }`, NOT `process.env`, so a child-process CLI drive whose flags live only in `runCli`'s env argument (e.g. the `--json` sweep's `ATLAS_TEST_MODE`/`ATLAS_FAKE_PROVIDER`) activates the fake instead of reaching for the Keychain / live Gemini.
- The invoker **validates its params at the boundary** (`EgressInvokeParamsSchema.safeParse`) BEFORE any key resolution or transport. `runId` is a canonical `Ulid` (not any non-empty string — a traversal-shaped value is rejected here). A malformed `maxTokens`/`dimensions`/`runId`, an extra field, or a mismatched operation/request shape throws a terminal `ProviderCallError{ kind:"validation" }` that never reaches the adapter (no transmission ⇒ no receipt/ledger row).
- `resolveGeminiApiKey(env?)` — `ATLAS_GEMINI_API_KEY` (env wins) else `security find-generic-password -s atlas-gemini-api-key -w`; **throws** `ProviderCallError{ kind:"authentication" }` when neither is present. Reads the supplied `env` mapping (default `process.env`). Call it ONLY on a real provider path, LAZILY. `hasGeminiApiKey(env?)` — the non-throwing presence probe (swallows a missing `security` binary / missing item), consumed by `status`. Constants: `GEMINI_API_KEY_ENV`, `GEMINI_KEYCHAIN_SERVICE`.
- `GenerateObjectClientRequest<T>` carries BOTH the caller's Zod `schema` (types `T`, re-validates locally) AND the `schemaId` string — only the id crosses the invoke seam.
- `modelCallId(runId, requestHash)` = `mc_` + first 32 hex of `sha256(runId \0 requestHash)` — the deterministic idempotency key. `buildModelCallStatement` → `INSERT ... ON CONFLICT(call_id) DO NOTHING`.

## Invariants & guardrails

- **Lazy credential resolution.** The key resolves ONLY on the first provider call inside `createInProcessInvoker`, never at import/process start — the blank-Ubuntu CI canary (no `security`, no env var) stays green on non-provider commands (`test/key-resolution.test.ts`).
- **No broker dependency in runtime code.** The adapter/types/errors/prompt-registry are DEFINED here, not re-exported. `! grep -rn '@atlas/broker\|@atlas/scan' packages/models/src` passes.
- **Schema-identity fail-closed** (`generateObject`): rejects a caller whose Zod `schema` is not **reference-identical** to the one registered for `schemaId` in `SCHEMA_REGISTRY` (a test may inject an overlay). Unknown `schemaId` → same error. Throws `ProviderCallError{ kind:"validation", retryable:false }` with **no transmission**. Both the client and the invoke side resolve the SAME schema for an id.
- **Exactly one receipt per transmission** (D6/D18): every success, refusal, OR provider error hands its receipt to the sink BEFORE the method returns/throws. A `model_calls` row is written for refused/errored transmissions too (tokens/cost actually consumed; 0 for a pre-flight refusal).
- **Pre-abort never transmits**: an already-aborted call throws `ProviderCallError{ kind:"cancelled" }` with **zero round-trips and no receipt**. An in-flight abort still yields an `error`-outcome receipt.
- **`model_calls` is a plain operational row (v2 #338).** The §2.8 audit-ledger protocol is retired — no `finalizeLedgerWrite`, no audit event, no per-run receipt journal. The CLI (`query.ts`) folds ALL a run's receipts into ONE plain `applyLedgerWrite` transaction alongside the `agent_runs` + `retrieval_*` rows (D6: many transmissions, one run, N rows, no per-call event).
- **Idempotent per `(runId, requestHash)`** via the derived `call_id` + `ON CONFLICT DO NOTHING`. A pre-serialization failure (e.g. `model_incompatible` at `serialize`) has **no serialized bytes to hash**, so it carries a deterministic "unbound" hash over the COMPLETE attempted request (`sha256("unbound:" + runId + JSON.stringify(body))`) — NOT just `runId/operation/model`, so two distinct failed calls in one run never collapse to a single `model_calls` row.
- **Audit record cannot drift from the receipt contract:** `ModelCallAuditRecordSchema` is DERIVED from the SSOT `ModelCallReceiptSchema` (`.omit({runId}).extend({callId}).strict()`). `ModelCallReceiptSchema` is a **survivor** — `model_calls` persistence depends on it.
- **`embed` batch semantics** (provider-interface §3): N vectors in input order; result `dimensions` MUST equal the requested value or it's a `validation` error (dimension drift opens a new index generation, D7). A `partial_batch` error names `succeededIndices`; a partial batch is **never persisted as complete**.

## Gotchas & sharp edges

- **The provider path carries NO capability + NO declared sensitivity.** `EgressInvokeParams` is `{ runId, body }`. The capability-only types (`EgressCapability`/`EgressLimits`/`EgressOperation`/`CapabilitySensitivity`) and `mintEgressCapability` are **dropped** — they died with the wrapper. The capability-secret env var that used to gate outbound calls is **gone** from the provider path; only the lazy Gemini key remains.
- **The v2 exit set is `{0,1,2,4,5}`.** Provider retryability rides `ProviderCallError.retryable` + `retryAfterMs` (never a code), which the jobs runner consumes — and the `jobs run` batch aggregate IS the one process path that can return the nominal `7`. `query` maps a retryable provider/embed failure to **exit 4** (`internal`) carrying `retryable`/`retryAfterMs` (the `embedding-retryable` envelope); `index rebuild` expresses a non-converged partial as **exit 2** (`config`) with per-item `retryable` flags.
- **`retryable` is FIXED per error kind, never caller-chosen** — `providerError()` computes it: retryable = NOT (`validation`|`authentication`|`cancelled`|`model_incompatible`). `quota` → `retryable:true`. `retryAfter` is present only for `rate_limit`/`quota`, propagated into both `retryAfter` and the CLI alias `retryAfterMs`.
- **`model_calls.operation` semantic label needs a caller override.** The invoke surface has 3 operations, but the ledger domain is `generate|extract|classify|synthesize|embed`. `buildModelCallStatement` defaults to the structural `OPERATION_MAP`; the semantic label is a per-call `operation` override on the opts.
- **What `generateText().text` contains is decided by the adapter's `extractText`** — Gemini thinking responses interleave `thought:true` parts; the adapter drops them so raw reasoning never releases (`src/gemini.ts`, PR #149 lineage). `finishReason` is surfaced so callers can detect truncation (#211).
- **The `thoughtSignature` history (ADR-0001, now superseded by ADR-0003)** is why the pre-cutover egress scanned RELEASED bytes, not the raw envelope — moot now that there is no in-process scan on the provider path, but the receipt still keeps `responseHash` = sha256 of the RAW provider bytes.
- **The gated fake provider (`ATLAS_TEST_MODE=1` + `ATLAS_FAKE_PROVIDER=1`)** swaps the real adapter for a deterministic in-process fake (hash embeddings; text ops return `"ok"`; `generateObject` ⇒ `{}`). `ATLAS_FAKE_PROVIDER_MODE` adds INERT-by-default per-case steering for the agentic E2E (`valid`/`malformed`/`schema-invalid`/`error`) — read ONLY when the fake is already active. STRICTLY test-only: both `ATLAS_TEST_MODE` + `ATLAS_FAKE_PROVIDER` must be `"1"` AND no explicit adapter/transport/resolver supplied (`src/client.ts`).
