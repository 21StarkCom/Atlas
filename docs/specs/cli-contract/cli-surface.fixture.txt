Atlas V1 ‚Äî CLI surface inventory (prose contract)
=================================================

This is the human-readable inventory of the `brain` command surface. It is a
CONTRACT: `contract-lint` parses every command line below and asserts a bijection
with the registry rows in `commands.json` ‚Äî a command here without a registry row
(or a registry row without a line here) fails the lint.

Parse rule: a command line is one that BEGINS with a backtick; the command name
is the text inside that first pair of backticks. Every other line ‚Äî this prose,
headings, and comments (lines starting with a hash) ‚Äî is ignored, even if it
contains backticked words. The text after the em dash is descriptive only.

# Health & diagnostics (Phase 1)
`inspect` ‚Äî read-only structural summary of the vault + store.
`doctor` ‚Äî environment/permission/lock/backup health checks; `--reclaim-locks` for dead-pid locks.
`status` ‚Äî read-only run/job/quarantine/backup/audit summary (Tier-0; emits run.readonly).

# Database / ledger (Phase 1)
`db status` ‚Äî projection + ledger + migration status.
`db verify` ‚Äî invariant checks over the store (with `--backup` to verify a backup file).
`db migrate` ‚Äî apply pending DDL migrations (idempotent, checksum-guarded).
`db rebuild` ‚Äî transactional rebuild of the vault-projection tables from Markdown.
`db backup` ‚Äî take an encrypted ledger backup (advances the watermark; `--force-unblock` recovery).
`db restore` ‚Äî privileged, destructive restore of ledger tables from a backup.

# Ingest & sources (Phase 2)
`ingest` ‚Äî capture + extraction/classification preview by default; `--apply` performs it.
`source add` ‚Äî deterministic Tier-1 source capture; applies immediately.
`source list` ‚Äî paginated list of captured sources.
`source show` ‚Äî show one source capture + renditions.
`source trust show` ‚Äî show a source's trust state.

# Jobs queue (Phase 2)
`jobs list` ‚Äî paginated list of queue jobs.
`jobs run` ‚Äî claim and drain queued jobs synchronously (single-runner lock).
`jobs retry` ‚Äî re-queue failed jobs.
`jobs cancel` ‚Äî cancel pending/claimed/running jobs.

# Notes (Phase 2)
`note add` ‚Äî deterministic Tier-1 authored-note ingest; applies immediately.
`note show` ‚Äî show a note by id or slug.
`note related` ‚Äî paginated related-notes traversal.
`note history` ‚Äî paginated note change history.

# Git workflow ‚Äî read/maintenance (Phase 2)
`git status` ‚Äî paginated open-run status.
`git cleanup` ‚Äî prune abandoned agent branches/worktrees (convergent).

# Retrieval (Phase 3)
`query` ‚Äî grounded question answering with citations.
`index status` ‚Äî index generation + staleness summary.
`index verify` ‚Äî verify index consistency.
`index repair` ‚Äî convergent index repair.
`index rebuild` ‚Äî full deterministic index rebuild from Markdown.

# Workflows ‚Äî mutation loop (Phase 4)
`enrich` ‚Äî enrich a note; preview by default, `--apply` mutates per risk tier.
`reconcile` ‚Äî alias/duplicate resolution; preview by default, `--apply` mutates.
`maintain` ‚Äî orphans/broken-links/stale maintenance; preview by default, `--apply` mutates.
`validate` ‚Äî deterministic vault validation checks.

# Git workflow ‚Äî review gate & privileged integration (Phase 4)
`git review` ‚Äî show a review-pending run's diff.
`git refresh` ‚Äî privileged: rebase + regenerate a superseded run (new commit + manifest).
`git approve` ‚Äî privileged: integrate an exact reviewed commit.
`git reject` ‚Äî reject a review-pending run (convergent).
`git rollback` ‚Äî privileged: roll back an integrated run.
`git verify` ‚Äî convergent manifest‚Üîindex reconciliation repair.

# Evidence & claims (Phase 4)
`evidence review` ‚Äî review flagged evidence/claims.
`evidence resolve` ‚Äî resolve a flagged evidence issue (convergent).
`evidence retry` ‚Äî retry evidence re-verification (convergent).

# Trust (Phase 4)
`source trust promote` ‚Äî privileged: promote a source's trust level.
`source trust revoke` ‚Äî privileged: revoke a source's trust.

# Erasure (Phase 4)
`purge` ‚Äî privileged, broker-authorized selector-based erasure across all storage classes.

# Graduation & quarantine (Phase 5)
`graduation scan` ‚Äî fail-closed full-vault (+ history) secret scan on the disposable copy.
`graduation audit` ‚Äî read-only bootstrap audit of legacy notes by quarantine category.
`graduation migrate` ‚Äî privileged: deterministic, review-gated bootstrap migration.
`quarantine inspect` ‚Äî privileged: inspect quarantined items.
`quarantine resolve` ‚Äî privileged: resolve a quarantined item.
`index eval` ‚Äî retrieval-quality eval (recall@10 / MRR) against a labeled query set; the graduation gate.

# Live vault sync (Phase 5)
`sync` ‚Äî run one upstream‚Üírefs/atlas/main absorb cycle over the adopted vault.
`sync reset` ‚Äî privileged: re-converge refs/atlas/main to upstream after a divergence halt.
`sync status` ‚Äî read the sync cursor, behind-by, pending quarantine, and divergence state.

# Console enablement (Phase 6)
`watch` ‚Äî long-lived NDJSON stream of jobs, model calls, ledger appends, backup watermark, daemon health.
